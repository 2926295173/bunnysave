# bunnysave-clone (省钱兔)

A faithful Next.js 15 clone of [bunnysave.com](https://www.bunnysave.com/) based on the
[bunnysave-tech-analysis.md](../bunnysave-tech-analysis.md) report.

## Tech stack

| Layer        | Choice                              | Notes                                      |
| ------------ | ----------------------------------- | ------------------------------------------ |
| Framework    | **Next.js 15.5** (App Router)       | RSC + streaming by default                 |
| Language     | TypeScript 5.7                      | strict mode                                |
| UI           | Tailwind CSS 3.4 + CSS Modules      | CSS variable holders mirror original style |
| Fonts        | `next/font/google` (Plus Jakarta Sans + Noto Sans SC) | self-hosted, preload |
| Images       | `next/image` (WebP, responsive srcset) | local assets in `public/images`           |
| State        | React Context (SessionProvider, SiteConfigProvider) | mirrors original components |
| Analytics    | Google Analytics (optional via env) | `NEXT_PUBLIC_GA_ID`                        |
| Data         | Static JSON in `data/` (scraped from bunnysave.com) | re-generable via `pnpm parse` |

## Pages

| Route                          | Purpose                                           |
| ------------------------------ | ------------------------------------------------- |
| `/`                            | Hero featured deal + grid + brand grid + newsletter |
| `/category/[slug]`             | Daily deals / Freebies / Coupons                  |
| `/deal/[id]`                   | Deal detail with related deals                    |
| `/search?q=`                   | Client-side fuzzy match                           |
| `/sitemap-info`, `/legal/*`    | Static info pages                                 |
| `/api/newsletter`              | POST endpoint that persists emails to `data/newsletter.json` |
| `/sitemap.xml`, `/robots.txt`  | Auto-generated by Next                            |

## Getting started

```bash
# 1. install
pnpm install

# 2. (optional) re-scrape data
python3 scripts/scrape.py          # downloads homepage.html + 21 images
.venv/bin/python scripts/parse.py  # builds data/deals.json + brands.json

# 3. dev
pnpm dev   # http://localhost:3000

# 4. production build
pnpm build
pnpm start
```

Requires Node 20+, pnpm 10+, and Python 3.11+ (for the scrape scripts).

## Data pipeline

`scripts/scrape.py` fetches `https://www.bunnysave.com/` and:

1. saves the SSR HTML to `data/homepage.html`
2. walks every `/_next/image?url=...` reference
3. downloads every `assets.dealselected.com/{deals,brands,site}/*` image into
   `public/images/<bucket>/<file>`
4. extracts deal candidates from the RSC stream

`scripts/parse.py` parses the SSR body with BeautifulSoup and emits clean
`data/deals.json` (10 deals) and `data/brands.json` (9 logos).

`pnpm parse` rebuilds the JSON from the cached HTML.

## Performance posture

Matches the original site per the analysis report:

- **RSC streaming** — every page is a server component that streams its payload.
- **Font preload** — `next/font` automatically emits `<link rel=preload>` for WOFF2 subsets.
- **Image optimization** — `<Image>` emits responsive srcset + lazy loading; WebP via `formats: ["image/webp"]`.
- **Selective hydration** — only `SearchBar` / `NewsletterForm` are client components.
- **Code splitting** — `optimizePackageImports` shrinks the React runtime.
- **SEO** — `hreflang` (zh/en/x-default), OG, Twitter cards, canonical, JSON-LD-friendly metadata.

## Deployment

Designed for Vercel.

### Option A — Git integration (recommended)

1. Push this repo to GitHub/GitLab/Bitbucket.
2. Import the project on https://vercel.com/new.
3. Set `NEXT_PUBLIC_GA_ID` (optional) in **Project Settings → Environment Variables**.
4. Vercel auto-builds on push. No tokens are stored in the repo.

### Option B — Manual deploy via REST API

```bash
# Get a token at https://vercel.com/account/settings/tokens (issue + revoke here)
export VERCEL_TOKEN=...   # never commit this
export VERCEL_TEAM_ID=...
python scripts/deploy.py
```

The script will refuse to run if `VERCEL_TOKEN` or `VERCEL_TEAM_ID` are missing.
Tokens are never hardcoded into the source tree.

### Going live checklist

- [ ] Update `src/lib/site.ts` SITE constants to match your real domain
- [ ] Configure custom domain in Vercel project settings
- [ ] Set `NEXT_PUBLIC_GA_ID` (or leave blank to disable analytics)
- [ ] Generate and set `AUTH_SECRET` (`openssl rand -base64 32`)
- [ ] (optional) Add Google OAuth via `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`
- [ ] Provide a hosted database via `AUTH_DB_URL` (Turso / Postgres) — required on Vercel
- [ ] Wire Magic Link emails via `RESEND_API_KEY` (otherwise links print to server logs in dev)

## Authentication

Three providers backed by **Neon Postgres** (`@neondatabase/serverless`):

- **Credentials** — email + password (bcrypt, 12 rounds)
- **Google OAuth** — click Google button on the auth page
- **Magic Link** — passwordless email login (15 min TTL), via `/api/auth/magic/request`

The logged-in session is reachable everywhere via `useSession()` from `next-auth/react`.
The Header top-right shows the avatar with a dropdown (账户设置 / 我的收藏 / 退出登录)
once authenticated; otherwise a “登录” pill that links to `/login`.

### Database setup

1. In the Vercel dashboard → Storage → **Create Database → Neon** (Postgres).
2. Vercel auto-injects `DATABASE_URL` on integration.
3. Locally run `vercel env pull .env.development.local` so dev points at the same DB
   (or your local Postgres).
4. Schema bootstrap runs lazily on the first auth request — no migrations needed.
   Tables created: `users`, `magic_links`.

### Magic Link emails

Set `RESEND_API_KEY` (free tier at https://resend.com) and `MAGIC_FROM`.
Without those, the magic link prints to the Vercel function logs as a
dev-only fallback.
