# bunnysave-clone (省钱兔)

A faithful Next.js 16 App Router clone of [bunnysave.com](https://www.bunnysave.com/). RSC-first, statically rendered from scraped JSON, with NextAuth v5 + Neon Postgres for auth.

## Tech stack

| Layer        | Choice                              | Notes                                      |
| ------------ | ----------------------------------- | ------------------------------------------ |
| Framework    | **Next.js 16** (App Router)         | RSC + streaming by default                 |
| Runtime      | Bun 1.3                             | package manager & runner                   |
| Language     | TypeScript 5.7                      | strict mode                                |
| UI           | Tailwind CSS 3.4                    | custom bunny palette                       |
| Fonts        | `next/font/google` (Plus Jakarta Sans + Noto Sans SC) | self-hosted, preload |
| Images       | `next/image` (WebP, responsive srcset) | local assets in `public/images`           |
| State        | React Context (SessionProvider, SiteConfigProvider) | mirrors original components |
| Data         | Neon Postgres + `src/data/*.json`   | deals/brands/categories live in DB        |
| Auth         | NextAuth v5                         | Credentials + Magic Link + Google OAuth    |

## Getting started

```bash
# 1. install
bun install

# 2. set up database (copy .env.local.keep → .env.local and fill in DATABASE_URL)
cp .env.local.keep .env.local

# 3. seed the database (required before first dev/build)
bun db:seed

# 4. dev
bun dev   # http://localhost:3000

# 5. production build
bun run build
bun start
```

Requires Bun 1.3+, Python 3.11+ (for scrape/parse scripts).

## Data pipeline

`scripts/scrape.py` fetches `https://www.bunnysave.com/` and:

1. saves the SSR HTML to `data/homepage.html`
2. downloads deal/brand/site images into `public/images/<bucket>/`
3. extracts deal candidates from the RSC stream

`scripts/parse.py` parses the SSR body with BeautifulSoup and emits `data/deals.json` and `data/brands.json`.

```bash
python3 scripts/scrape.py          # raw scrape
.venv/bin/python scripts/parse.py  # build JSON
bun db:seed                        # populate Postgres
```

## Database

Neon Postgres via `@neondatabase/serverless`. Tables:

- **Auth**: `users`, `magic_links`
- **Catalog**: `categories`, `brands`, `deals`, `deal_categories`
- **Admin**: `deal_submissions`, `audit_log`

Schema bootstrap runs lazily on first call — no migrations needed. Seed with `bun db:seed`.

## Authentication

Three providers backed by Neon Postgres:

- **Credentials** — email + password (bcrypt, 12 rounds)
- **Google OAuth** — conditionally enabled via `GOOGLE_CLIENT_ID`/`GOOGLE_CLIENT_SECRET`
- **Magic Link** — passwordless email login (15 min TTL)

The logged-in session is reachable everywhere via `useSession()` from `next-auth/react`.

### Setup

1. Vercel: Storage → Neon integration auto-injects `DATABASE_URL`
2. Local: `cp .env.local.keep .env.local` — fill in your Neon connection string
3. Magic link emails: set `RESEND_API_KEY` and `MAGIC_FROM`, or links print to console

## Deployment

Designed for Vercel. Git integration recommended — push to GitHub, import on Vercel, set env vars.

### Env vars

| Variable              | Required | Notes                                    |
| --------------------- | -------- | ---------------------------------------- |
| `DATABASE_URL`        | yes      | Neon Postgres connection string          |
| `AUTH_SECRET`         | yes      | `openssl rand -base64 32`                |
| `GOOGLE_CLIENT_ID`    | no       | Google OAuth                             |
| `GOOGLE_CLIENT_SECRET`| no       | Google OAuth                             |
| `RESEND_API_KEY`      | no       | Magic link email delivery                |
| `MAGIC_FROM`          | no       | Magic link sender address                |
| `ADMIN_EMAIL`         | no       | Admin user email (for `ensure-admin.ts`) |
| `ADMIN_PASSWORD`      | no       | Admin user password                      |
